# 掃描目錄按鈕 - 設定檢查設計

## 問題

目前按下「掃描目錄」按鈕時，會直接彈出 `prompt()` 讓使用者輸入路徑，**完全忽略**「模型根目錄」的設定。若使用者未在設定頁面填寫路徑，掃描功能也能正常運作，但無法利用已儲存的捷徑。

## 目標

當使用者按下「掃描目錄」按鈕時：
1. 先檢查「模型根目錄」是否已有設定值
2. 若有 → 直接使用該路徑執行掃描（不再詢問）
3. 若無 → 開啟設定 Modal + Toast 提示，引導使用者填寫

## 邏輯流程

```
[點擊掃描按鈕]
      │
      ▼
[讀取 model_root_path 設定]
      │
      ├─── 已設定 ──→ [直接掃描該路徑]
      │
      └─── 未設定 ──→ [Toast: "請先設定模型根目錄"]
                       [開啟設定 Modal]
```

## 修改範圍

### `frontend/js/main.js`
- 修改 `btn-scan` 點擊事件
- 加入 `settingsAPI.get("model_root_path")` 檢查
- 已設定路徑：直接呼叫 `modelsAPI.scan(directory)`
- 未設定路徑：開啟設定 Modal + Toast

## 實作細節

### 流程 A（有設定）
- 直接以 `model_root_path` 作為掃描目錄
- 錯誤處理與原本相同

### 流程 B（無設定）
- Toast 訊息：「請先設定模型根目錄」
- 自動開啟設定 Modal（`openSettings()` 函式已存在）
- 使用者填寫並儲存後，可再次點擊掃描

## 成功標準

- [ ] 已設定模型根目錄時，點擊掃描按鈕直接執行掃描（不再彈 prompt）
- [ ] 設定儲存後，掃描功能正常運作
